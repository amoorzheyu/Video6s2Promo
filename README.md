## Video6s2Promo：六秒产品宣传短视频生成工具

一个基于 **FastAPI + React/Vite** 的小工具，用来批量生成 5 段、每段约 6 秒的产品宣传视频（共约 30 秒），支持：

- **上传产品参考图**（如【图一】所示，必须是产品图，不是视频帧）
- **5 个片段标题 & 文案可完全自定义编辑**
- **一次性并行生成 5 段视频**，全部成功后再自动拼接为总视频
- **单段视频可二次“重新生成”**，并自动更新总视频
- **默认提示词内置【产品】【人物1】【画面要求】结构**，方便你在前端自由修改

---

## 功能概览

- **多片段脚本编辑**
  - 每个片段都有「标题」与「内容」。
  - 「内容」整段会原样发送给视频生成模型，你可以完全修改。
  - 默认文案遵循脚本格式：  
    - 开头包含：`【产品】...`、`【人物1】...`、`【画面要求】...`  
    - 后面按时间点描述：`0~1s：...`、`1~2s：...` 等。

- **产品 & 人物约束（可编辑）**
  - **产品**：在 `【产品】` 中精确描述产品（磁吸小夜灯），包括：外观、颜色、长宽高、光效、感应头、磁吸底座等，并注明“如【图一】所示”。
  - **人物1**：在 `【人物1】` 中描述中国妈妈的年龄、脸型、发型、衣着（颜色与款式）等，保证 5 段中的人物形象一致。
  - **画面要求**：在 `【画面要求】` 中约束物理逻辑（重力、光线方向、阴影、动作轨迹），禁止产品漂浮、乱飘等不符合物理的画面。

- **视频生成逻辑**
  - 前端点击「生成」后，后端会：
    1. 使用你上传的产品图作为 **所有片段的参考图**；
    2. 根据当前 5 段文案，**并行调用外部视频生成接口**；
    3. 每个片段生成完成后即可单独预览；
    4. 当 5 个片段都完成后，再把它们 **拼接成一个总视频**。

- **单片段重新生成**
  - 当 5 段都已生成完成后，你可以针对某一段点击「重新生成」：
    - 后端只会重新生成该段视频；
    - 原有总视频会在该段成功后自动重新拼接；
    - 前端通过版本号参数强制刷新浏览器缓存。

- **播放器行为**
  - 总视频与单片段预览 **默认打开声音**，但 **不会自动播放**，需要手动点击播放。

## 技术栈

- **前端**
  - React 18 + TypeScript
  - Vite
  - Axios（与后端通信）
  - 组件：`SegmentEditor`、`PreviewPanel`、`ProgressTracker`、`SizeSelector` 等

- **后端**
  - Python 3 + FastAPI
  - `httpx`（调用外部视频生成 API）
  - `asyncio` / `asyncio.gather`（并发生成 5 段视频）
  - `aiofiles`（异步文件 IO）
  - `imageio-ffmpeg`（拼接视频）
  - `python-dotenv`（环境变量管理）

---

## 目录结构（简要）

- `backend/`
  - `main.py`：FastAPI 入口
  - `models.py`：任务状态 `TaskState`
  - `routers/video.py`：视频相关接口（生成、状态查询、获取视频、重新生成）
  - `services/generator.py`：调用外部视频生成 API、下载视频
  - `services/prompts.py`：5 段默认脚本文案（含【产品】【人物1】【画面要求】）
- `frontend/`
  - `src/App.tsx`：主界面逻辑
  - `src/constants/defaultSegments.ts`：前端 5 段默认文案
  - `src/components/SegmentEditor.tsx`：片段标题与文案编辑（Tab 形式切换）
  - `src/components/PreviewPanel.tsx`：视频预览 & 重新生成按钮
  - `src/components/ProgressTracker.tsx`：并行生成进度显示
  - `src/components/SizeSelector.tsx`：视频尺寸比例选择

## 后端启动

1. 进入后端目录：

```bash
cd backend
```

2. 准备虚拟环境并安装依赖（示例）：

```bash
python -m venv .venv
.venv\Scripts\activate  # Windows PowerShell
pip install -r requirements.txt
```

3. 配置环境变量（`.env`）：

```bash
cp .env.example .env  # 如果有示例文件
```

在 `.env` 中填入外部视频生成服务的 Key 等信息（例如 Grok imagine video 相关配置）。

4. 启动 FastAPI（开发模式）：

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

---

## 前端启动

1. 进入前端目录：

```bash
cd frontend
```

2. 安装依赖：

```bash
npm install
```

3. 启动开发服务器：

```bash
npm run dev
```

4. 浏览器访问 Vite 提示的地址（通常是 `http://localhost:5173`）。

> 如需与后端联调，请确认前端请求的 API 地址与 FastAPI 实际监听地址一致（端口等可在 `.env` 或配置文件中调整）。

## 使用流程示例

1. **上传产品参考图**
   - 上传一张清晰的产品图（磁吸小夜灯），作为所有片段的 `【图一】`。

2. **确认/修改每段标题与文案**
   - 在「片段标题与文案」区域，通过 Tab 切换 1~5 段。
   - 可以直接修改默认内容中的：
     - `【产品】...`
     - `【人物1】...`
     - `【画面要求】...`
     - 以及后面的 `0~1s：...`、`1~2s：...` 等时间脚本。

3. **选择视频尺寸**
   - 在尺寸选择中选择需要的比例（例如：`9:16`、`1:1` 等），仅显示当前选中项的高亮边框。

4. **点击生成**
   - 前端会调用 `/api/generate`，后端会并行生成 5 段视频。
   - 生成过程中：
     - 右侧/下方进度区域会显示 5 段都在“生成中”；
     - 上方预览区域会在某段先完成后，立即可以播放该段。

5. **查看总视频**
   - 当 5 段都完成后，后端会自动拼接成一个总视频；
   - 前端会出现总视频入口，可点击播放（默认有声但不自动播放）。

6. **单段重新生成（可选）**
   - 如果对某个片段不满意，可点击该缩略图上的「重新生成」按钮；
   - 等待该段重新生成完成后，总视频会自动更新。

## 提示词格式建议

- **推荐结构（可自由修改，仅作为默认模板）：**

```text
【产品】...（详细说明产品外观/尺寸/用法，如【图一】所示）
【人物1】...（人物年龄、脸型、发型、衣服颜色与款式等）
【画面要求】...（物理逻辑、光线方向、禁止漂浮等）

0~1s：...
1~2s：...
2~3s：...
3~4s：...
4~5s：...
5~6s：...
```

- **说明**
  - 所有约束都写在可编辑的文本里，**后端不会强行拼接额外规则**；
  - 你可以按自己的习惯增删 `【产品】` / `【人物1】` / `【画面要求】` 的内容；
  - 台词建议自然、口语化，只需保证大致能在对应时间内说完，不再强制“约 X 字/秒”。

## 已知特性与注意事项

- 外部视频生成服务存在一定不确定性，同样提示词可能产生略有差异的结果；
- 物理逻辑与人物一致性由提示词引导，模型可能仍有少量偏差，可通过多次「重新生成」优化；
- 如果接口返回空视频 URL，后端内置了简单重试机制，仍失败时会在任务状态中记录错误信息。
